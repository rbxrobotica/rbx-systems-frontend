## 3. Frequência e calendário sugerido

- **2 artigos técnicos por semana** (terças e quintas).
- **1 journal/post estratégico por semana** (segundas).
- **1 case a cada duas semanas** (sextas, alternadas).
- **1 update de produto por mês**.
- **1 repost adaptado para LinkedIn** por conteúdo publicado.

**Ciclo de 90 dias:**

- Dias 1–30: clusters RBX Systems + Leandro Damasio + IA/LLMOps.
- Dias 31–60: clusters Financeiro + Logística + Produtos.
- Dias 61–90: cases, changelog, journal e conteúdo de autoridade avançada.

---

## 4. Formato dos posts

Cada artigo deve conter:

1. Título otimizado (< 60 caracteres).
2. Meta description única (< 160 caracteres).
3. H1 claro.
4. Estrutura de headings (H2, H3).
5. Introdução com problema/tse.
6. Corpo técnico com exemplos ou modelos.
7. Conclusão com CTA.
8. Links internos (3–5 por post).
9. Schema JSON-LD (`BlogPosting`, `Article`, `Service`, `SoftwareApplication`).
10. Versão curta para LinkedIn.

---

## 5. Versões para LinkedIn

Para cada artigo, produzir:

- Gancho (1 frase).
- Resumo do argumento (2–3 parágrafos).
- CTA com link para o post completo.
- Hashtags: #LLMOps #AIEngineering #AgenticAI #RBXSystems #LeandroDamasio
